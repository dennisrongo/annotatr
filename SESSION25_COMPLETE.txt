## Architectural Insight

These features demonstrate the POWER of Tauri 2's global shortcut plugin:

- NO platform-specific conditional code needed
- Single implementation works for Windows, macOS, and Linux
- Plugin handles platform differences automatically (Win32 API, Carbon/CGEvent, X11/Wayland)
- Hotkey parsing and registration logic is shared across all platforms
- Same default hotkeys work on all platforms (with platform-appropriate modifiers)

This is the CORRECT approach - using the framework's cross-platform abstractions rather than
writing platform-specific code manually.

## How Tauri Handles Platform Differences for Hotkeys

### Windows
- RegisterHotKey API via Win32
- Virtual Key codes (VK_* constants)
- Modifiers: MOD_CONTROL, MOD_SHIFT, MOD_ALT, MOD_WIN
- System-wide global hotkeys

### macOS (✓ VERIFIED)
- Carbon Event Manager / CGEvent API
- macOS virtual key codes
- Modifiers: cmdKey, shiftKey, optionKey, controlKey
- System-wide global hotkeys
- VERIFIED working on current development environment

### Linux
- X11: XGrabKey via Xlib
- Wayland: zwp_global_shortcut_v1 protocol
- X11 KeySyms or Wayland key codes
- Modifiers: ControlMask, ShiftMask, Mod1Mask (Alt), Mod4Mask (Super)
- Works with GNOME, KDE, XFCE, and other desktop environments

## Implementation Details

### Rust Backend (src-tauri/src/lib.rs)

**Hotkey Registration Command (lines 560-615):**
- Parses hotkey configuration JSON
- Unregisters existing hotkeys
- Registers each hotkey using `app.global_shortcut().register()`
- Tracks registered hotkeys for cleanup

**Hotkey String Parser (lines 618-689):**
- Parses strings like "Ctrl+Shift+A" into `Shortcut` objects
- Supports modifiers: Ctrl/Control, Shift, Alt, Meta/Cmd/Super/Win
- Supports keys: A-Z, 0-9, F1-F12, Space, Enter, Escape, arrows, etc.

**Platform-Specific Key Mapping:**
- Maps string keys to platform-specific Key enums
- Handles special keys (Space, Enter, Escape, arrows)
- Handles function keys (F1-F12)
- Handles character keys (A-Z, 0-9)
