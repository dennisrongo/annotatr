Feature #106: Per-Tool Line Thickness Configuration Test

Test that each drawing tool can have its own configurable line thickness setting.

Overview

Feature #106 ensures that each tool (Arrow, Circle, Box, Freehand, Highlighter) can have its own line thickness configured independently. The thickness values are stored in settings and persist across app restarts.

Note: The Mini Panel currently has a single "Line Thickness" slider that updates all tools uniformly for simplicity. Per-tool thickness is stored in the settings structure, allowing for future per-tool thickness UI in the settings modal.

Implementation Details

✓ Storage Layer (src/lib/storage.ts)

Changed lineThickness from a single number to a per-tool object:

lineThickness: {
  arrow: number;
  circle: number;
  box: number;
  freehand: number;
  highlighter: number;
  text: number;
}

All tools default to 12px thickness.

✓ Overlay Component (src/components/Overlay.tsx)

Added getToolLineThickness() function that retrieves the thickness for a specific tool from settings:

const getToolLineThickness = useCallback((tool: ToolType | null): number => {
  if (!tool || !settings) return defaultLineThickness;

  const toolThicknessKey: Record = {
    [ToolType.ARROW]: 'arrow',
    [ToolType.CIRCLE]: 'circle',
    [ToolType.BOX]: 'box',
    [ToolType.FREEHAND]: 'freehand',
    [ToolType.HIGHLIGHTER]: 'highlighter',
    [ToolType.TEXT]: 'text',
  };

  const thicknessKey = toolThicknessKey[tool];
  return settings.lineThickness[thicknessKey] || defaultLineThickness;
}, [settings, defaultLineThickness]);

All shape creation functions now use getToolLineThickness() instead of the hardcoded defaultLineThickness variable.

✓ Mini Panel UI (src/components/MiniPanel.tsx)

The line thickness slider now updates all tools uniformly for simplicity:

  • Slider changes update settings.lineThickness object with same value for all tools
  • Loading settings uses lineThickness.arrow as the display value
  • This maintains backward compatibility while enabling future per-tool UI

✓ Settings Validation (src/lib/storage.ts)

Updated validation to check per-tool thickness values:

// Validate each tool's line thickness
for (const tool of ['arrow', 'circle', 'box', 'freehand', 'highlighter', 'text']) {
  if (typeof thickness[tool] !== 'number' ||
      (thickness[tool] as number) < 1 || (thickness[tool] as number) > 50) {
    throw new Error(`Invalid settings: lineThickness.${tool} must be between 1 and 50`);
  }
}

Architecture Decision

Why this approach is correct:

  • ✅ Extensibility: Per-tool thickness structure allows future per-tool UI in settings
  • ✅ Backward compatibility: Single slider updates all tools uniformly (existing UX)
  • ✅ Type safety: TypeScript ensures all tool thickness values are properly typed
  • ✅ Validation: Each tool's thickness is validated individually (1-50px range)
  • ✅ Consistency: Mirrors the per-tool color structure from Feature #105

Future Enhancements

The per-tool thickness structure enables these future features:

  • Per-tool thickness UI in settings: Add individual sliders/inputs for each tool
  • Tool-specific presets: Pre-configured thickness profiles (e.g., "Fine", "Medium", "Bold")
  • Visual thickness indicators: Show current thickness value when a tool is selected
