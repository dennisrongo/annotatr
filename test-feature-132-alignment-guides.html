Feature #132: Shape Alignment Guides

PASSING

Show visual alignment guides when drawing shapes for better precision

Feature Overview

Feature #132 adds visual alignment guides that appear when drawing shapes, helping users create precise, aligned annotations. The guides appear automatically when shapes align with:

  • Center of screen - Horizontal and vertical center lines
  • Screen edges - Top, bottom, left, and right edges
  • Existing shapes - Centers and key points of previously drawn shapes
  • Snapping - Shapes automatically snap to alignment positions

Implementation Details

1. Detection Functions

Three new functions detect alignment opportunities:

// Detect alignment with center of screen const detectCenterAlignment = useCallback((x: number, y: number) => { const centerX = canvas.width / (2 * devicePixelRatio); const centerY = canvas.height / (2 * devicePixelRatio); return { horizontal: Math.abs(x - centerX) < SNAP_THRESHOLD, vertical: Math.abs(y - centerY) < SNAP_THRESHOLD, }; }, []); // Detect alignment with edges of screen const detectEdgeAlignment = useCallback((x: number, y: number) => { const width = canvas.width / devicePixelRatio; const height = canvas.height / devicePixelRatio; return { left: Math.abs(x) < SNAP_THRESHOLD, right: Math.abs(x - width) < SNAP_THRESHOLD, top: Math.abs(y) < SNAP_THRESHOLD, bottom: Math.abs(y - height) < SNAP_THRESHOLD, }; }, []); // Detect alignment with existing shapes const detectShapeAlignment = useCallback((x: number, y: number) => { const alignments = []; shapesRef.current.forEach(shape => { // Calculate shape center based on type // Check horizontal and vertical alignment // Add to alignments array if within threshold }); return alignments; }, []);

2. Guide Rendering

Visual guides are drawn as dashed blue lines with snap indicators:

const drawAlignmentGuides = useCallback((ctx, guides) => { ctx.save(); // Set guide style ctx.strokeStyle = "#00BFFF"; // Deep sky blue ctx.lineWidth = 1; ctx.setLineDash([5, 5]); // Dashed line ctx.globalAlpha = 0.7; // Draw horizontal/vertical center guides // Draw edge guides // Draw shape alignment guides // Draw snap point indicator (small circle) ctx.restore(); }, []);

3. Snapping Behavior

When alignment is detected, the shape position automatically snaps:

// Apply snapping to center let snappedX = currentX; let snappedY = currentY; if (centerAlign.horizontal) { snappedY = canvas.height / (2 * devicePixelRatio); } if (centerAlign.vertical) { snappedX = canvas.width / (2 * devicePixelRatio); } // Snap to shape alignments shapeAligns.forEach(guide => { if (guide.type === 'shape-horizontal') { snappedY = guide.y; } else if (guide.type === 'shape-vertical') { snappedX = guide.x; } });

Notes

DPI Awareness

All alignment calculations use window.devicePixelRatio to ensure guides work correctly on high-DPI displays (Retina, 4K, etc.). The canvas dimensions are divided by the device pixel ratio to get CSS pixel coordinates.

Performance Considerations

Alignment detection runs on every mouse move during drawing. The checks are lightweight (simple distance calculations) and don't iterate through all shapes on every frame - only checking shape centers which is O(n) where n is the number of shapes.

Visual Clarity

Guides use a dashed line pattern with reduced opacity (0.7) to ensure they're visible but don't obscure the drawing. The deep sky blue color (#00BFFF) provides good contrast against most backgrounds.

Tool Exclusions

Freehand and Highlighter tools don't show alignment guides because they're continuous drawing tools rather than point-to-point shapes. Text tool also doesn't use guides since text placement is click-based.
