Feature #128: Per-Shape Fade Duration Override

Allow individual shapes to have custom fade durations

✓ PASSING

Overview

This feature allows users to specify a custom fade duration for individual shapes, independent of the global fade duration setting. This is useful when you want certain shapes to persist longer or shorter than others.

Key Behaviors:

  • One-time use: Custom duration applies only to the next shape drawn, then resets to global setting
  • Per-shape override: Each shape stores its own customFadeDuration value
  • Shape independence: Different shapes can have different fade durations on screen simultaneously
  • Visual indicator: Active custom duration shows green "ACTIVE" badge
  • Auto-reset: After drawing a shape, custom duration automatically turns off

All 6 shape creation functions updated to include custom duration:

5. Auto-Reset After Shape Creation

Custom duration resets to null after each shape is created:

Auto-fade system now checks each shape for custom duration:

Technical Details

Shape Data Structure:

Each shape now optionally includes a customFadeDuration property. If present, it overrides the global fadeDuration setting for that specific shape. The value is stored in seconds (not milliseconds) for consistency with the global setting.

Priority Resolution

Scenario Duration Used
Shape has customFadeDuration set Use customFadeDuration (e.g., 20s)
Shape has no customFadeDuration Use global fadeDuration from settings (e.g., 10s)
Custom duration checkbox unchecked All new shapes use global setting
After shape is drawn with custom duration Next shape uses global setting (auto-reset)

Event Flow

User checks checkbox → MiniPanel state updated ↓ emit("custom-fade-duration", 20) ↓ Overlay event listener fires ↓ setCustomFadeDuration(20) ↓ User draws shape → Shape creation includes customFadeDuration: 20 ↓ setCustomFadeDuration(null) ← Auto-reset ↓ Next shape uses global setting
