### Feature #123: Undo Last Shape ✓

**Status:** PASSING (Implemented)

**Implementation Summary:**

Added ability to undo the most recently created shape using standard undo keyboard shortcut:
- Ctrl+Z on Windows/Linux
- Cmd+Z (⌘+Z) on macOS

**Code Changes:**

1. **undoLastShape Function** (src/components/Overlay.tsx, lines 207-230):
   ```tsx
   const undoLastShape = useCallback(() => {
     if (shapesRef.current.length === 0) {
       console.log("[Undo] No shapes to undo");
       return;
     }

     // Remove the last shape
     const removedShape = shapesRef.current.pop();

     // Remove opacity tracking for the undone shape
     if (removedShape && shapeOpacities[removedShape.id]) {
       const newOpacities = { ...shapeOpacities };
       delete newOpacities[removedShape.id];
       setShapeOpacities(newOpacities);
     }

     console.log(`[Undo] Removed shape: ${removedShape?.tool}`);
     console.log(`[Undo] Remaining shapes: ${shapesRef.current.length}`);

     // Redraw the canvas
     redrawAllShapes();
   }, [shapeOpacities, redrawAllShapes]);
   ```

2. **Keyboard Handler** (src/components/Overlay.tsx, lines 638-643):
   ```tsx
   // Feature #123: Handle Ctrl+Z / Cmd+Z to undo last shape
   if ((event.ctrlKey || event.metaKey) && event.key === "z") {
     event.preventDefault();
     console.log("Undo hotkey pressed (Ctrl+Z / Cmd+Z)");
     undoLastShape();
     return;
   }
   ```

**Key Behaviors:**
- Removes most recent shape (LIFO - Last In, First Out)
- Cleans up opacity tracking to prevent memory leaks
- Updates canvas immediately via redrawAllShapes()
- Works with all 6 tool types
- Cross-platform support (Ctrl+Z and Cmd+Z)
- Graceful handling when no shapes exist
- Can be used multiple times to undo multiple shapes

**Bug Fix:**
Fixed TypeScript compilation error by adding `panelTransparency: 0.95` to fallback settings in Overlay.tsx (line 955).

**Verification:**

## Technical Notes

### Feature #114 Key Implementation Detail

**Cancellation Logic:**
The key insight is that we need to check `drawingState.isDrawing` BEFORE the existing overlay dismissal logic. This allows Escape to have dual behavior:
- While drawing: Cancel current shape
- Not drawing: Dismiss overlay

This preserves the existing overlay dismissal behavior while adding the new cancellation capability.

### Feature #115 Verification

This feature was already fully implemented in previous sessions (Features #105 and #106). The verification confirmed:
- All defaults are properly defined in DEFAULT_SETTINGS
- Helper functions (getToolColor, getToolLineThickness) work correctly
- All shape creation functions apply the defaults
- No code changes were needed

### Feature #123 Implementation Highlights

**LIFO Order:**
Using `Array.pop()` naturally provides LIFO behavior, which is the expected undo pattern in most applications.

**Memory Management:**
Critical to clean up `shapeOpacities` for undone shapes to prevent memory leaks. If opacity entries accumulate, they would cause the state object to grow indefinitely.

**Platform Detection:**
Using both `event.ctrlKey` (Windows/Linux) and `event.metaKey` (macOS) ensures cross-platform compatibility without explicit platform detection code.
