Toggle State Flow

The toggle hotkey (Ctrl+Shift+D by default) intelligently switches between two states:

Drawing Mode OFF
⇅ Press Ctrl+Shift+D ⇅
Drawing Mode ON

State Transition Logic:

  • Current State: OFF → Press toggle → New State: ON
  • Current State: ON → Press toggle → New State: OFF

Implementation Details

Backend Command (Rust)

/// Feature #62: Toggle drawing mode on/off /// Shows overlay and enables drawing mode if disabled, or dismisses overlay if enabled #[tauri::command] fn toggle_drawing_mode(app: AppHandle) -> Result { let overlay_window = app.get_webview_window("overlay") .ok_or("Overlay window not found")?; // Check current state by inspecting overlay visibility let is_visible = overlay_window.is_visible()?; if is_visible { // Overlay is visible - dismiss it (turn OFF drawing mode) dismiss_overlay(app)?; println!("Drawing mode toggled OFF (overlay dismissed)"); Ok(false) } else { // Overlay is hidden - show it and enable drawing mode (turn ON) show_overlay(app)?; // Update state to enable drawing mode if let Some(state) = app.state::().try_get() { let mut state_guard = state.lock().map_err(|e| format!("State lock error: {}", e))?; state_guard.is_visible = true; state_guard.drawing_mode = true; } // Emit drawing mode enabled event app.emit("drawing-mode-changed", true) .map_err(|e| format!("Failed to emit event: {}", e))?; println!("Drawing mode toggled ON (overlay shown, drawing mode enabled)"); Ok(true) } }

How It Works

Activation (OFF → ON)

  1. User presses Ctrl+Shift+D while drawing mode is OFF
  2. Global shortcut triggers toggle_drawing_mode command
  3. Command checks overlay visibility: is_visible() → false
  4. Calls show_overlay() to display overlay
  5. Updates SharedState: is_visible = true, drawing_mode = true
  6. Emits drawing-mode-changed event with value true
  7. Frontend receives event and updates UI
  8. Drawing mode is now ON

Deactivation (ON → OFF)

  1. User presses Ctrl+Shift+D while drawing mode is ON
  2. Global shortcut triggers toggle_drawing_mode command
  3. Command checks overlay visibility: is_visible() → true
  4. Calls dismiss_overlay() to hide overlay
  5. Updates SharedState: is_visible = false, drawing_mode = false
  6. Emits events to clear shapes and reset cursor
  7. Frontend receives events and cleans up UI
  8. Drawing mode is now OFF
