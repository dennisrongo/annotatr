src-tauri/src/lib.rs:900-934
/// Dismiss the overlay (hide and clean up state) /// Feature #20: This is called when Escape key is pressed or toggle hotkey is triggered /// Properly deactivates overlay and cleans up existing shapes #[tauri::command] fn dismiss_overlay(app: AppHandle) -> Result<(), String> { // Get the overlay window by label let overlay_window = app.get_webview_window("overlay") .ok_or("Overlay window not found")?; // Update state to mark overlay as not visible if let Some(state) = app.state::().try_get() { let mut state_guard = state.lock().map_err(|e| format!("State lock error: {}", e))?; state_guard.is_visible = false; // Feature #20: Also disable drawing mode when dismissing state_guard.drawing_mode = false; } // Hide the overlay window overlay_window.hide()?; // Disable mouse capture (return to pass-through mode) overlay_window.set_ignore_cursor_events(true)?; // Feature #20: Emit event to clear drawing state and shapes app.emit("overlay-dismissed", ()) .map_err(|e| format!("Failed to emit overlay-dismissed event: {}", e))?; // Feature #20: Clear all shapes app.emit("clear-all-shapes", ()) .map_err(|e| format!("Failed to emit clear-all-shapes event: {}", e))?; // Feature #20: Emit drawing mode changed event to reset cursor app.emit("drawing-mode-changed", false) .map_err(|e| format!("Failed to emit drawing-mode-changed event: {}", e))?; println!("Overlay dismissed - shapes cleared, drawing state reset"); Ok(()) }

What this command does:

  • Hides the overlay window
  • Updates shared state (is_visible = false, drawing_mode = false)
  • Disables mouse capture (returns to pass-through mode)
  • Emits "overlay-dismissed" event for UI cleanup
  • Emits "clear-all-shapes" event to remove all drawn shapes
  • Emits "drawing-mode-changed" event to reset cursor

Verification Steps

To verify that Feature #68 is working correctly, follow these steps:

  1. Launch Annotatr
    Start the application using npm run tauri dev
  2. Enter Drawing Mode
    Click any tool button (Arrow, Circle, Box, Freehand, Highlighter, Text) in the mini panel
    OR press a hotkey (e.g., Ctrl+Shift+A for Arrow)
  3. Verify Overlay is Visible
    The overlay window should appear and capture mouse input
    The cursor should change to indicate drawing mode
  4. Start Drawing (optional)
    Click and drag to start drawing a shape (e.g., start drawing an arrow)
  5. Press Escape Key
    While in drawing mode (with or without an active drawing), press the Escape key
  6. Verify Cancellation
    ✓ Overlay window should immediately hide
    ✓ Any active drawing should be cancelled
    ✓ All shapes should be cleared from the overlay
    ✓ Mouse capture should be disabled (click-through restored)
    ✓ Drawing mode state should be set to false
    ✓ Cursor should return to normal
  7. Check Console Logs
    In the overlay window console, you should see:
    "Escape key pressed - dismissing overlay"
    "Overlay dismissed successfully"
  8. Repeat Test
    Test the Escape key multiple times with different tools to ensure consistent behavior
Note: The Escape key works at any time during drawing mode:
• Before starting to draw (overlay visible, no shapes)
• While actively drawing (mouse button down, shape in progress)
• After drawing (shapes visible on overlay)

In all cases, pressing Escape dismisses the overlay and clears all drawing state.
