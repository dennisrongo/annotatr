Easing Function Details

The implementation uses an ease-out cubic function: 1 - (1 - t)³

Fade Timeline (for 10-second fade duration):
• 0-8 seconds: Full opacity (1.0)
• 8-10 seconds: Smooth fade from 1.0 to 0.0
• 10+ seconds: Shape removed from canvas

Why ease-out cubic?

  • Starts fading slowly (smooth transition from full opacity)
  • Accelerates toward the end (quick fade to transparent)
  • Provides professional, polished appearance
  • Common in UI/UX design for dismissible elements

Technical Details

Easing Function Code

const getFadeOpacity = useCallback((age: number, fadeDuration: number): number => { // Start fading when shape is 80% through its lifetime const fadeStart = fadeDuration * 0.8; const fadeEnd = fadeDuration; const fadeDurationActual = fadeEnd - fadeStart; // Before fade starts, full opacity if (age < fadeStart) { return 1.0; } // After fade ends, zero opacity if (age >= fadeEnd) { return 0.0; } // During fade, use ease-out cubic: 1 - (1 - t)^3 const t = (age - fadeStart) / fadeDurationActual; const opacity = 1 - Math.pow(1 - t, 3); // Invert opacity (1.0 -> 0.0) return 1 - opacity; }, []);

Opacity Calculation Flow

1. Every 50ms, calculate age of each shape 2. Apply getFadeOpacity() to get opacity value (0.0-1.0) 3. Update shapeOpacities state with new values 4. redrawAllShapes() passes opacities to drawShape() 5. drawShape() applies ctx.globalAlpha = opacity 6. Shape renders with calculated opacity 7. If opacity <= 0, shape is removed from array

Performance Considerations

  • Update frequency: 50ms = 20 fps (smooth but not CPU-intensive)
  • State updates: Only when shapes exist (empty check could optimize further)
  • Memory: Opacity entries cleaned up when shapes removed
  • Rendering: Canvas API handles alpha blending efficiently
