Feature #71: Per-Shape Fade Timer Started on Creation

Category: Auto-Fade System
Description: Start an individual fade timer for each shape when it is created

Implementation Summary

Overview

Each shape is assigned a unique createdAt timestamp when created, which serves as its individual fade timer. The auto-fade system checks each shape's age against this timestamp to determine when it should be removed.

How It Works

Shape Created → createdAt = Date.now()
↓
Shape Added to shapesRef[]
↓
Auto-Fade Loop (every 1s)
↓
For Each Shape:
  age = now - shape.createdAt
  if (age > fadeDuration) → Remove Shape

Architecture & Design

Timestamp-Based Timer Approach

The implementation uses a timestamp-based approach rather than explicit setTimeout/Timer objects. This is more efficient and prevents timer management issues.

Advantages:

  • ✓ Memory efficient: No timer objects per shape
  • ✓ Scalable: Works with unlimited shapes
  • ✓ Robust: No timer cleanup needed on shape removal
  • ✓ Simple: Single setInterval checks all shapes
  • ✓ Accurate: Millisecond precision timestamps
