What is Real-time Preview?

Real-time preview means that when a user changes a setting, they can immediately see how that change affects the application. The UI updates instantly to reflect the new value, allowing users to preview changes before committing to them.

Benefits of Real-time Preview:

  • Immediate Feedback: Users see changes instantly without waiting
  • Experimentation: Users can try different values and see results immediately
  • Reduced Errors: Users can correct mistakes before finalizing
  • Better UX: No need to save and reload to see changes

6. Hotkey Editor (Settings Modal)

Location: src/components/MiniPanel.tsx (lines 411-453)
⚠️ Different Behavior - Explicit Save Required

Hotkeys require an explicit "Save" button click because:

  • Global hotkeys must be unregistered before registering new ones
  • Registration is a system-level operation (can't be done on every keystroke)
  • Prevents conflicts from partial key combinations

Capture Handler:

const handleHotkeyCapture = (e: KeyboardEvent) => {
  e.preventDefault();
  e.stopPropagation();

  // Build hotkey string from pressed keys
  const modifiers = [];
  if (e.ctrlKey) modifiers.push("Ctrl");
  if (e.shiftKey) modifiers.push("Shift");
  if (e.altKey) modifiers.push("Alt");
  if (e.metaKey) modifiers.push("Cmd");

  const key = e.key.toUpperCase();
  const hotkeyString = [...modifiers, key].join("+");

  setEditingHotkey(hotkeyString);  // ← Immediate preview in input field
};

Save Handler:

const saveHotkey = async () => {
  try {
    // Update the hotkey in the config
    const updatedHotkeys = {
      ...hotkeys,
      [editingHotkeyName]: editingHotkey,
    };
    setHotkeys(updatedHotkeys);

    // Save to persistent storage
    const settings = await loadSettings();
    settings.hotkeys = updatedHotkeys as any;
    await saveSettings(settings);

    // Re-register all hotkeys with the new configuration
    await invoke("register_hotkeys", { hotkeyConfig: settings });

    // Check for conflicts
    const conflictsResult = await invoke("check_hotkey_conflicts", {
      hotkeyConfig: settings,
    });

    if (conflictsResult.has_conflicts) {
      setHotkeyConflicts(conflictsResult.conflicts);
    }

    setEditingHotkeyName(null);
    setEditingHotkey("");
    console.log("Hotkey saved and re-registered:", editingHotkeyName, editingHotkey);
  } catch (error) {
    console.error("Failed to save hotkey:", error);
  }
};
Preview Behavior:
  • User clicks "Edit" → enters edit mode
  • User presses keys → input field shows captured combination immediately
  • User can see the hotkey string in real-time (e.g., "Ctrl+Alt+A")
  • User clicks "Cancel" → reverts to original hotkey
  • User clicks "Save" → hotkey is registered and persisted

How Real-time Preview Works

React State Management

The key to real-time preview is React's state management. When state changes, React automatically re-renders components that use that state.

// 1. State declaration
const [lineThickness, setLineThickness] = useState(12);

// 2. State is bound to UI
<input value={lineThickness} onChange={handleChange} />

// 3. Handler updates state
const handleChange = (value) => {
  setLineThickness(value);  // Triggers re-render
  saveSettings({ lineThickness: value });  // Persists to disk
};

// 4. Component re-renders with new state
// Slider's value prop updates to show new value
// Any other component using lineThickness also updates

Immediate Feedback Loop

User Action (drag slider/click color)
        ↓
onChange Event Fires
        ↓
Handler Function Called
        ↓
setState(newValue)  ← React state updates immediately
        ↓
Component Re-renders ← UI shows new value instantly
        ↓
saveSettings() ← Persists to disk (also immediate)
        ↓
User sees changes in real-time
