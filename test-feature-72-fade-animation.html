Feature #72: Fade Animation for Each Shape Individually

Category: Auto-Fade System
Description: Implement fade animation that applies to each shape independently based on its timer

Implementation Summary

Overview

Smooth fade-out animations are implemented using opacity transitions. Each shape's opacity is calculated independently based on its age and the configured fade duration. The animation uses an ease-out cubic function for smooth deceleration during the fade-out phase.

Animation Phases

Shape Lifecycle (with 10s fade duration)
Phase 1: Visible
0-8 seconds
Opacity: 100%
Phase 2: Fading
8-10 seconds
Opacity: 100% → 0%
Phase 3: Removed
10+ seconds
Opacity: 0%

Easing Function Details

Why Ease-Out Cubic?

The ease-out cubic function (1 - (1 - t)³) provides a smooth, natural fade-out effect:

  • Start of fade: Fast transition (immediately visible)
  • Middle of fade: Decelerating (smooth transition)
  • End of fade: Very slow (gentle disappear)

Timing Configuration

Phase Duration Percentage Purpose
Visible 8s (of 10s) 80% User sees full shape
Fading 2s (of 10s) 20% Smooth transition to invisible

Configurability: The 80% threshold is defined in getFadeOpacity() and can be adjusted if needed.

Performance Characteristics

Update Frequency

  • Interval: 50ms (20 frames per second)
  • Purpose: Smooth animation without excessive CPU usage
  • Comparison: 60fps = 16.67ms, 30fps = 33.33ms, 20fps = 50ms

Per-Shape Operations

Each update cycle (every 50ms):

  • Iterate through all shapes (O(n))
  • Calculate age for each shape
  • Compute opacity using easing function
  • Update shapeOpacities state
  • Redraw canvas with new opacities

Efficiency Optimizations

  • ✓ Only redraw when opacities change
  • ✓ Clean up faded shapes immediately
  • ✓ Use useCallback for memoization
  • ✓ Single setState() per cycle (batched updates)
