How It Works

  1. Startup Registration: On app launch, MiniPanel loads settings and calls register_hotkeys
  2. Tauri Global Shortcut: Uses tauri-plugin-global-shortcut for system-wide hotkeys
  3. Hotkey Press: When user presses Ctrl+Shift+[Tool Key], the registered callback fires
  4. Tool Activation: Callback calls activate_tool_hotkey with the tool name
  5. Overlay Show: If overlay is hidden, it's shown and focused
  6. Event Emission: "tool-selected" event is emitted to overlay with tool type
  7. State Update: Overlay updates its current tool state

Feature #104: Tool Switching While in Drawing Mode

Implementation Location

Frontend: src/components/MiniPanel.tsx Lines: 295-310 (selectTool function - works regardless of state) Backend: src-tauri/src/lib.rs Lines: 861-894 (activate_tool_hotkey - no drawing state check)

How It Works

Key Design Decision: The tool selection system does NOT check if drawing is currently in progress. This allows users to switch tools at any time, even mid-draw.

  1. No State Check: selectTool function doesn't check for active drawing
  2. Immediate Switch: Tool state changes immediately when button/hotkey triggered
  3. Event Propagation: Tool-selected event is emitted to overlay immediately
  4. Drawing State: Overlay component handles tool switch - may cancel current shape or complete it
  5. Smooth UX: Users can change their mind about tool choice at any time

Why This Approach?

User Experience Benefits

  • Flexibility: Users aren't locked into a tool once selected
  • Quick Corrections: If user starts drawing with wrong tool, they can switch immediately
  • Workflow: Power users can pre-switch tools while moving mouse to next drawing location
  • No Blocking: No need to complete or cancel current shape before switching
