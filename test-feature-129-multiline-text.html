Feature #129: Multi-line Text Support

✓ PASSING - Implemented

Overview

Feature #129 adds support for multi-line text shapes in the Annotatr application. Users can now create text annotations that span multiple lines, making it easier to add detailed notes, instructions, or comments to screen recordings.

Implementation Details

1. Text Input Component (Overlay.tsx)

Change: Replaced <input> with <textarea>

<textarea ref={textInputRef} value={textInputValue} onChange={(e) => setTextInputValue(e.target.value)} onKeyDown={handleTextInputKeyDown} onBlur={handleTextInputSubmit} style={{ position: "absolute", left: textInputPosition.x, top: textInputPosition.y, fontSize: settings ? `${Math.round(settings.fontSize * 1.7)}px` : "24px", padding: "4px 8px", border: `2px solid ${settings?.colors.text || "#FF0000"}`, borderRadius: "4px", outline: "none", backgroundColor: "rgba(255, 255, 255, 0.9)", color: "#000000", minWidth: "200px", minHeight: "60px", // NEW: Allow multiple lines zIndex: 10000, resize: "both", // NEW: User can resize the textarea overflow: "auto", // NEW: Scrollbars when content exceeds size }} placeholder="Type multi-line text and press Enter to submit..." />

Key Changes:

  • minHeight: "60px" - Shows multiple lines by default
  • resize: "both" - Users can resize the textarea horizontally and vertically
  • overflow: "auto" - Scrollbars appear when content exceeds the visible area
  • Updated placeholder - Now mentions "multi-line text"

2. Keyboard Handler (Overlay.tsx)

Change: Updated handleTextInputKeyDown to support Shift+Enter for new lines

const handleTextInputKeyDown = useCallback((event: React.KeyboardEvent<HTMLTextAreaElement>) => { if (event.key === "Enter" && !event.shiftKey) { // Enter submits the text event.preventDefault(); handleTextInputSubmit(); } else if (event.key === "Escape") { // Escape cancels text input event.preventDefault(); setTextInputVisible(false); setTextInputValue(""); } // Shift+Enter allows default behavior (new line) }, [handleTextInputSubmit]);

Key Behavior:

  • Enter - Submits the text and creates the shape
  • Shift+Enter - Creates a new line (default textarea behavior)
  • Escape - Cancels text input without creating a shape
  • Click outside (onBlur) - Submits the text

3. Multi-line Rendering (drawing.ts)

Change: Updated drawText function to render multiple lines

export function drawText( ctx: CanvasRenderingContext2D, shape: TextShape ): void { const { position, text, color, fontSize } = shape; ctx.fillStyle = color; ctx.font = `${fontSize}px sans-serif`; ctx.textBaseline = "top"; // Feature #129: Handle multi-line text const lines = text.split('\n'); const lineHeight = fontSize * 1.2; // 20% line spacing lines.forEach((line, index) => { const y = position.y + (index * lineHeight); ctx.fillText(line, position.x, y); }); }

Key Implementation:

  • text.split('\n') - Splits text into array of lines
  • lineHeight = fontSize * 1.2 - 20% spacing between lines
  • forEach loop - Renders each line at the appropriate Y offset
  • Y offset calculation - position.y + (index * lineHeight)
