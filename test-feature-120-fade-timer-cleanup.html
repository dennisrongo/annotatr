Feature #120: Fade Timer Cleanup

Properly clean up fade timers when shapes are manually removed or app closes

✓ PASSING

Overview

This feature ensures that opacity tracking entries are properly cleaned up when shapes are removed from the canvas, preventing memory leaks. The cleanup happens in multiple scenarios:

  • Undo operation - When a user undoes the last shape (Ctrl+Z)
  • Overlay dismissed - When the overlay window is closed/dismissed
  • Clear all shapes - When all shapes are cleared via event
  • Monitor change - When switching monitors (shapes filtered)
  • Component unmount - When the Overlay component unmounts
  • App shutdown - When the app closes (beforeunload event)

Implementation Summary

Centralized Cleanup Function

A new cleanupOpacityTracking() function was added to handle all opacity cleanup:

/** * Feature #120: Cleanup opacity tracking for specific shape IDs * Prevents memory leaks when shapes are removed */ const cleanupOpacityTracking = useCallback((shapeIds: string | string[]) => { const idsToRemove = Array.isArray(shapeIds) ? shapeIds : [shapeIds]; if (idsToRemove.length === 0) return; const newOpacities = { ...shapeOpacities }; let cleanedCount = 0; idsToRemove.forEach(id => { if (newOpacities[id]) { delete newOpacities[id]; cleanedCount++; } }); if (cleanedCount > 0) { setShapeOpacities(newOpacities); console.log(`[Feature #120] Cleaned up opacity tracking for ${cleanedCount} shape(s)`); } }, [shapeOpacities]);

Memory Leak Prevention

Why This Matters:

Before Feature #120, when shapes were removed manually (undo, clear, dismiss), their opacity tracking entries remained in the shapeOpacities state object. This caused a memory leak because:

  • The state object would accumulate orphaned entries indefinitely
  • Each entry consumed memory even though the corresponding shape was gone
  • In long-running sessions, this could lead to significant memory usage
  • The auto-fade interval would continue processing non-existent shapes

Feature #120 ensures cleanup happens in all removal scenarios, preventing memory leaks.
