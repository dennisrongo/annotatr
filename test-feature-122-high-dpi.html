Implementation Details

1. DPI Detection (Step 1)

Location: src/components/Overlay.tsx (line ~1044)

Implementation: 
const dpr = window.devicePixelRatio || 1;
How it works:
  • window.devicePixelRatio returns the ratio of physical pixels to CSS pixels
  • Standard displays: devicePixelRatio = 1
  • Retina displays: devicePixelRatio = 2 (or higher)
  • High-DPI Windows displays: devicePixelRatio = 1.25, 1.5, 1.75, etc.

2. Canvas Scaling (Step 2)

Location: src/components/Overlay.tsx (lines ~1047-1053)

Implementation: 
// Set display size (css pixels)
canvas.style.width = window.innerWidth + "px";
canvas.style.height = window.innerHeight + "px";

// Set actual size in memory (scaled to account for extra pixel density)
canvas.width = window.innerWidth * dpr;
canvas.height = window.innerHeight * dpr;
How it works:
  • CSS size: Controls how large the canvas appears on screen (CSS pixels)
  • Internal size: Controls how many pixels are actually stored in memory
  • For Retina (DPR=2): A 1000x1000 CSS pixel canvas becomes 2000x2000 physical pixels
  • This ensures every CSS pixel maps to multiple physical pixels for sharp rendering

3. Coordinate Transformation (Step 3)

Location: src/components/Overlay.tsx (line ~1056)

Implementation: 
// Feature #122: Normalize coordinate system to use css pixels
ctx.scale(dpr, dpr);
How it works:
  • ctx.scale(dpr, dpr) scales the canvas context by the device pixel ratio
  • All drawing operations can use CSS pixel coordinates directly
  • The context automatically maps CSS pixels to physical pixels
  • Mouse coordinates from events are already in CSS pixels, so no transformation needed
Example: 
// Before: Had to manually multiply coordinates
ctx.fillRect(10 * dpr, 10 * dpr, 100 * dpr, 100 * dpr);

// After: Can use CSS pixels directly
ctx.fillRect(10, 10, 100, 100);

Technical Explanation

Why DPI Scaling is Needed

The Problem:

On high-DPI displays, one CSS pixel corresponds to multiple physical pixels. Without DPI scaling:

  • The canvas has too few pixels to fill the screen sharply
  • The browser stretches the canvas, causing blurriness
  • Text and fine details appear pixelated
The Solution:

With DPI scaling implemented:

  • The canvas has enough physical pixels for the display
  • The context scales coordinates automatically
  • Rendering is sharp and crisp on all displays

Coordinate System Transformation

Key Insight: By using ctx.scale(dpr, dpr), we normalize the coordinate system:

  • All drawing code continues to use CSS pixel coordinates
  • The context automatically scales to physical pixels
  • Mouse events don't need transformation (they're already in CSS pixels)
  • The feature is transparent to existing drawing code

No changes needed to:

  • Arrow drawing (drawArrow)
  • Circle drawing (drawCircle)
  • Box drawing (drawBox)
  • Freehand drawing (drawFreehand)
  • Highlighter drawing (drawHighlighter)
  • Text drawing (drawText)
  • Mouse event handlers (handleMouseDown, handleMouseMove, handleMouseUp)
