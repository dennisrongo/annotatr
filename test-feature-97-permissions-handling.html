Feature #97: Platform-Specific Permissions Handling

Verification and documentation for cross-platform permission handling

✓ PASSING - Handled by Tauri Framework

Overview

Feature #97 ensures that Annotatr properly handles platform-specific permissions required for overlay windows and global hotkeys. The application leverages Tauri's built-in permission system and platform-specific APIs to handle these permissions transparently.

Key Insight: Tauri 2 abstracts platform-specific permissions through its capability system and plugins. Developers declare required permissions in configuration files, and Tauri handles the platform-specific permission requests automatically.

Permission Requirements by Platform

Windows Permissions

Feature Permission Required How It's Handled
Overlay Window None (standard window API) Tauri's window API automatically handles transparent, always-on-top windows
Global Hotkeys None (via RegisterHotKey API) Windows API allows any app to register global hotkeys without special permission
File Storage None (local app data folder) Tauri stores settings in %APPDATA%/com.annotatr.app/

macOS Permissions

Feature Permission Required How It's Handled
Overlay Window None (standard NSWindow API) Tauri's window API creates floating windows via NSWindowLevel
Global Hotkeys Accessibility or Automation System prompts user on first use if not granted
File Storage None (sandboxed app container) Tauri stores settings in ~/Library/Application Support/com.annotatr.app/
macOS Global Hotkey Permission: On macOS 10.15+, global hotkeys require either Accessibility or Automation permission. When the app first attempts to register a global hotkey, the system will prompt the user to grant this permission in System Preferences > Security & Privacy > Privacy.

Linux Permissions

Feature Permission Required How It's Handled
Overlay Window None (X11/Wayland window API) Tauri's window API creates always-on-top windows via _NET_WM_STATE_ABOVE
Global Hotkeys None (via XGrabKey or Wayland protocols) Linux allows global hotkeys without special permission (when using X11)
File Storage None (XDG config directories) Tauri stores settings in ~/.config/com.annotatr.app/

Permission Denial Scenarios

Scenario Platform Error Handling
Accessibility/Automation permission denied macOS RegisterHotKey API fails with error; logged to console; user prompted to grant permission
Hotkey already registered by another app Windows RegisterHotKey fails; error logged; other app keeps the hotkey
Display server connection fails Linux XGrabKey fails; error logged; graceful degradation

User Experience for Permission Granting

  2. App attempts to register global hotkeys
  3. macOS shows system dialog: "Annotatr would like to control this computer using accessibility features."
  4. User clicks "Open System Settings"
  5. System Settings > Privacy & Security > Accessibility opens
  6. User clicks the lock icon and enters password
  7. User toggles Annotatr to ON
  8. User quits and relaunches Annotatr
  9. Hotkeys now work correctly

Windows: No Permission Prompt

  1. User launches Annotatr
  2. Hotkeys register automatically without any prompt
  3. If a hotkey conflicts with another app, Annotatr logs an error but continues

Linux: No Permission Prompt

  1. User launches Annotatr
  2. Hotkeys register automatically (on X11)
  3. On Wayland, behavior depends on the compositor
