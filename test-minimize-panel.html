Feature #52: Panel Minimize/Hide Option

Feature Description

Add ability to minimize or hide the mini panel (e.g., when not needed during recording). Users can click a minimize button in the panel header to hide it from view.

Implementation Details

Backend Changes (src-tauri/src/lib.rs):

  • toggle_mini_panel() - Toggles panel visibility (hide/show)
  • hide_mini_panel() - Hides the mini panel window
  • show_mini_panel() - Shows and focuses the mini panel window

Frontend Changes (src/components/MiniPanel.tsx):

  • toggleMinimize() - Handler for minimize button
  • Added minimize button (−) in panel header
  • Button styled with hover effect
  • Positioned next to drag indicator

User Interface:

  • Minimize button (−) in panel header
  • Hover effect: color changes from #666 to #333
  • Tooltip: "Minimize panel (hide from view)"
  • Accessible with aria-label

Manual Testing Instructions

To test:
  1. Run the Tauri app: npm run tauri dev
  2. The mini panel window should open
  3. Locate the minimize button (−) in the panel header (next to ⋮⋮)
  4. Click the minimize button
  5. Expected: Mini panel window should hide
  6. To restore: Use system tray or global hotkey (if implemented)

Technical Notes

How It Works:

  1. User clicks minimize button (−) in panel header
  2. toggleMinimize() handler calls invoke("toggle_mini_panel")
  3. Backend toggle_mini_panel() command checks window visibility
  4. If visible: calls window.hide() to hide the panel
  5. If hidden: calls window.show() and window.focus() to restore
  6. Result: Panel window is hidden/shown without destroying it

Use Cases:

  • Hide panel during screen recording when not needed
  • Temporarily hide to view content behind panel
  • Clean up workspace while keeping app running

Future Enhancements:

  • System tray icon to restore hidden panel
  • Global hotkey to toggle panel visibility
  • Keyboard shortcut (e.g., Cmd/Ctrl+M) to minimize
