Feature #70: Configurable Fade Duration (default 10 seconds)

Category: Auto-Fade System
Description: Allow users to configure the duration before shapes automatically fade, defaulting to 10 seconds

Implementation Summary

Overview

Added a slider control in the Mini Panel that allows users to configure the auto-fade duration for shapes. The duration can be set from 1 to 60 seconds, with a default of 10 seconds. Settings are persisted to local storage and loaded on app startup.

Changes Made

1. src/components/MiniPanel.tsx
  • Added fadeDuration state variable (line 70)
  • Added loadFadeDuration() useEffect to load from storage (lines 185-199)
  • Added handleFadeDurationChange() function to save changes (lines 461-473)
  • Added fade duration slider UI control (lines 995-1056)
  • Updated Settings modal to show current fade duration (line 1140)

UI Control Details

The fade duration control includes:

  • Range: 1 to 60 seconds
  • Default: 10 seconds
  • Step: 1 second increments
  • Visual feedback: Current value displayed in blue (e.g., "10s")
  • Auto-save: Changes saved immediately to persistent storage
  • Placement: Below Font Size control in Mini Panel

Data Flow

User drags slider → handleFadeDurationChange() called → fadeDuration state updated → saveSettings() called → Storage writes to disk → Overlay reads on next check → Shapes fade after new duration
