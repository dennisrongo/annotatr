2. **Global Shortcut Plugin** (Cargo.toml)
```toml

```
- Handles platform-specific hotkey APIs automatically
- macOS: Triggers Accessibility/Automation permission prompt if needed
- Windows: No special permission required
- Linux: No special permission required (on X11)

3. **Error Handling** (src-tauri/src/lib.rs)
```rust
app.global_shortcut().register(shortcut, move || {
    // Callback
}).map_err(|e| format!("Failed to register hotkey '{}': {}", hotkey_value, e))?;
```
- Catches permission denials gracefully
- Logs errors to console
- App continues to function via mini panel if hotkeys fail

**Permission Requirements by Platform:**

| Feature | Windows | macOS | Linux |
|---------|---------|-------|-------|
| Overlay Window | None (standard API) | None (NSWindow) | None (X11/Wayland) |
| Global Hotkeys | None (RegisterHotKey) | **Accessibility/Automation** | None (XGrabKey) |
| File Storage | None (%APPDATA%) | None (sandboxed) | None (XDG config) |

**macOS Permission Flow:**
1. User launches Annotatr for first time
2. App attempts to register global hotkeys
3. macOS shows prompt: "Annotatr would like to control this computer using accessibility features"
4. User grants permission in System Preferences > Security & Privacy > Privacy > Accessibility
5. User quits and relaunches Annotatr
6. Hotkeys now work correctly

## Technical Notes

**Key Architectural Decision: Cross-Platform via Abstraction**

All three cross-platform features demonstrate the power of Tauri 2's abstraction layer:

1. **No Platform-Specific CSS:** Feature #96 uses only web-standard CSS with no platform-specific branches
2. **No Permission Code:** Feature #97 relies on Tauri plugins to handle platform-specific permissions
3. **No Build Configuration:** Feature #98 uses a single tauri.conf.json for all platforms

**Why This Approach is Correct:**

✓ **CORRECT:** Use Tauri 2's cross-platform abstraction layer
- Single codebase for all platforms
- Platform differences handled by Tauri framework
- Automatic updates when Tauri adds platform support
- Reduced maintenance burden
- Consistent behavior across platforms

✗ **INCORRECT:** Write platform-specific code manually
- Three separate code paths to maintain
- Risk of inconsistent behavior
- Reinventing the wheel (Tauri already solved this)
- Increased bug surface area
- Higher maintenance burden

**Industry Standard:**

This is how Electron, Tauri, and other cross-platform frameworks work. The entire point of using Tauri is to avoid writing platform-specific code.

### Key Architectural Decision

**✓ CORRECT:** Use Tauri 2's Unified Global Shortcut Plugin

The application uses `tauri-plugin-global-shortcut v2`, which provides a single, unified API for
registering global hotkeys across Windows, macOS, and Linux. This is the industry-standard approach
for cross-platform desktop applications.

**Benefits:**
- Single codebase for all platforms
- Platform differences handled by the plugin
- Automatic updates when Tauri adds platform support
- Reduced maintenance burden
- Consistent behavior across platforms

**Alternative (NOT used):** Manual platform-specific code using Win32 API on Windows, Carbon/CGEvent
on macOS, and X11/Wayland directly on Linux. This would require three separate code paths,
increasing complexity and bug surface area.
