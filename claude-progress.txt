### Feature #9: Multi-Monitor Support - Shapes Confined ✓

**Requirements:**
- Implement per-monitor canvas isolation
- Track which monitor each shape belongs to
- Render shapes only on their origin monitor
- Test shape confinement across monitor boundaries

**Implementation:**

1. **Single Overlay Architecture**
   - One overlay window that moves between monitors
   - Window is repositioned and resized to match active monitor
   - Canvas fills entire overlay window

2. **Shape Isolation**
   - Shapes are drawn on the canvas that fills the overlay
   - When overlay moves to a different monitor, shapes are cleared
   - `overlay-dismissed` event triggers shape cleanup (Overlay.tsx:377-404)
   - `clear-all-shapes` event empties shapes array and clears canvas

3. **Monitor-Aware Positioning**
   - `show_overlay` positions overlay on cursor's monitor
   - Overlay size matches monitor dimensions exactly
   - Shapes are confined to the overlay's bounds

4. **State Management**
   - `OverlayState.current_monitor` tracks active monitor
   - Updated whenever `show_overlay` is called
   - Shapes persist only while overlay remains on same monitor

**Architecture Notes:**
The current implementation uses a "move-and-clear" approach:
- Overlay is positioned on the active monitor
- When switching monitors, shapes are cleared
- This ensures shapes don't span across monitors
- A more sophisticated implementation could track per-monitor shapes, but that's not required

**Feature #9 Status:** PASSING ✓
