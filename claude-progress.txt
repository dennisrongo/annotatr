## Technical Notes

### Multi-line Text Implementation

**Line Spacing Formula:**
```typescript
const lineHeight = fontSize * 1.2; // 20% spacing
```

This provides readable spacing without excessive gaps. For a 24px font:
- Line height: 28.8px
- Line 1 Y: originalY
- Line 2 Y: originalY + 28.8px
- Line 3 Y: originalY + 57.6px

**Rendering Loop:**
```typescript
const lines = text.split('\n');
lines.forEach((line, index) => {
  const y = position.y + (index * lineHeight);
  ctx.fillText(line, position.x, y);
});
```

**Backward Compatibility:**
- Single-line text (without \n) splits into array with one element
- forEach loop runs once, rendering the single line
- No performance overhead for existing single-line shapes

```rust
#[tauri::command]
fn register_hotkeys(app: AppHandle, hotkey_config: serde_json::Value) -> Result<(), String> {
    unregister_all_hotkeys(app.clone())?;

    for (hotkey_name, hotkey_value) in hotkeys_config.as_object().unwrap() {
        let shortcut = Shortcut::new(hotkey_value.clone())?;
        let tool = hotkey_name.clone();

        app.global_shortcut().register(shortcut, move || {
            if let Err(e) = activate_tool_hotkey(app_handle.clone(), tool.clone()) {
                eprintln!("Failed to activate tool '{}': {}", tool, e);
            }
        })?;
    }
}
```

3. **Tool Activation (lib.rs:861-894)**
```rust
#[tauri::command]
fn activate_tool_hotkey(app: AppHandle, tool: String) -> Result<(), String> {
    let tool_type = convert_hotkey_tool_name(&tool);

    // Show overlay if not visible
    if !overlay_window.is_visible()? {
        overlay_window.show()?;
        overlay_window.set_focus()?;
        overlay_window.set_always_on_top(true)?;
        state_guard.drawing_mode = true;
    }

    // Emit tool-selected event to overlay
    app.emit("tool-selected", tool_type)?;
    app.emit("drawing-mode-changed", true)?;

    Ok(())
}
```

```tsx
const redrawAllShapes = useCallback(() => {
    const context = getCanvasContext();
    if (!context) return;

    const { ctx } = context;
    redrawShapes(ctx, shapesRef.current, shapeOpacities);
}, [getCanvasContext, shapeOpacities]);

// redrawShapes utility (from lib/drawing.ts)
export function redrawShapes(
    ctx: CanvasRenderingContext2D,
    shapes: Shape[],
    opacities?: Record<string, number>
): void {
    clearCanvas(ctx);
    shapes.forEach((shape) => {
        const opacity = opacities?.[shape.id];
        drawShape(ctx, shape, opacity);
    });
}
```

## Technical Notes

### Feature #122 Key Implementation Detail

**Canvas Scaling Pattern:**
```typescript
// Get DPI
const dpr = window.devicePixelRatio || 1;

// Set CSS size (how big it looks)
canvas.style.width = window.innerWidth + "px";
canvas.style.height = window.innerHeight + "px";

// Set physical size (how many pixels)
canvas.width = window.innerWidth * dpr;
canvas.height = window.innerHeight * dpr;

// Normalize coordinate system
ctx.scale(dpr, dpr);
```

This pattern ensures:
- CSS pixels control layout size
- Physical pixels control rendering sharpness
- Drawing code continues to use CSS pixels
- Mouse events work without transformation (already in CSS pixels)
