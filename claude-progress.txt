5. Settings persist across app restarts via `saveSettings()`

**Color Picker Architecture:**
- Main panel color picker (Features #44, #45): For quick color changes during drawing
- Settings modal color picker (Feature #81): For configuring default colors per tool
- Both use same state and storage system
- Both use same preset palette
- Both support custom color selection

**State Management:**
```typescript
// Per-tool color state
currentColorForTool: {
  arrow: "#FF0000",
  circle: "#FF0000",
  box: "#FF0000",
  freehand: "#FF0000",
  highlighter: "#FFFF00",  // Semi-transparent in practice
  text: "#FF0000"
}

// Currently selected color (for main panel)
selectedColor: "#FF0000"
```

**Persistence Flow:**
```
User Action → handleSettingsColorSelect()
    ↓
Update currentColorForTool state
    ↓
saveSettings({ colors: updatedColors })
    ↓
Tauri storage (settings.json)
    ↓
loadSettings() on next app startup
```

## Technical Notes

**Settings Controls Pattern:**

All three features follow the same consistent implementation pattern:

1. **State Variable:** `const [value, setValue] = useState(DEFAULT_SETTINGS.key)`
2. **Load Effect:** `useEffect` to load from storage on mount
3. **Change Handler:** `handleXChange` updates state and saves to storage
4. **UI Component:** Range slider with min/max/step attributes
5. **Value Display:** Bold blue text showing current value with unit
6. **Labels:** Min and max values shown on slider ends

**Storage Integration:**
```typescript
// Save to persistent storage
await saveSettings({ key: value });

// Load from persistent storage
const settings = await loadSettings();
setValue(settings.key);
```

**Real-Time Updates:**

All sliders use the `onChange` event for immediate updates:
```typescript
onChange={(e) => handleChange(parseInt(e.target.value, 10))}
```

No "Save" button required - changes persist immediately.
