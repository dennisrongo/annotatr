Persistence Behavior Documentation

When are Settings Saved?

  • Immediately: All sliders (line thickness, font size, fade duration) save on change
  • Immediately: Color picker saves on color selection
  • Explicitly: Hotkey changes save when user clicks "Save" button
  • On Reset: Reset button immediately writes defaults to storage

When are Settings Loaded?

  • On App Start: MiniPanel component useEffect loads settings on mount
  • After Reset: Frontend reloads settings from storage after reset
  • Merge Behavior: Loaded settings are merged with DEFAULT_SETTINGS to ensure all keys exist

Error Handling:

  • If storage fails to save, error is logged to console and user is alerted
  • If storage fails to load, default values are returned (app doesn't crash)
  • Save functions return Promise so errors can be caught by caller
